At QCF, you can learn various technologies not through only zoom calls or videos, but by building real time applications on daily basis.

This page is reachable by the URL - 'http://ec2-3-109-213-162.ap-south-1.compute.amazonaws.com/', which is not a very real website name because it was created for demonstration purpose only. This URL is just a Public IPv4 domain of an AWS EC2 instance. This text you are reading is being served by a Java based Sprint Boot application, which is deployed inside an AWS EC2 instance. Inside this Ec2, the spring boot application runs on a docker engine. The spring application exposes an endpoint which is exposed on port 8080 through an embeded tomcat server. Further docker maps the 8080 application port to port 80 of the EC2 instance. The security groups of EC2 are further configured to expose the response of endpoint to the internet. The whole (1 hour) exercise aims to enable a developer to learn and do following:

  • Develop a simple Java Spring Boot Application (10 mins)
  • Setup a GIT repo, clone it in IDE and add code. (5 mins)
  • Setup Maven (to build Java app) (5 mins)
  • Setup Docker (to run Java app)(5 mins)
  • Register/Setup a new AWS Account and familiarize with EC2 service (5 mins)
  • Create a new EC2 instance and connect to it using ssh from local system (15 mins)
  • Setup docker on EC2 instance (5 mins)
  • Build a docker image on local, push it to registry, pull inside EC2 and run the application (10 mins)
  • Expose spring application's rest endpoint, deployed on EC2, to internet using a public domain.
